# Accord — Trustless Deal Engine with AI Negotiation

Accord is an autonomous deal broker for freelance agreements built on Algorand blockchain. It solves the core trust problem: buyers fear poor delivery, sellers fear non-payment. Instead of relying on a centralized platform, Accord uses AI agents to negotiate the best deal and Algorand smart contracts to lock funds in escrow — no middleman ever holds the money.

Three AI seller agents compete for the buyer's task in a live negotiation chat. A scoring optimizer picks the best deal. The buyer funds an on-chain escrow via Pera Wallet. Then an AI delivery oracle autonomously evaluates whether the seller delivered — releasing funds to the seller or refunding the buyer automatically.

**The entire pipeline is AI-driven and trustless.** Intelligence is visible. Blockchain trust flow is visible.

---

## How It Works

1. **Connect Wallet** — Buyer connects Pera Wallet on Algorand TestNet
2. **Submit Task** — Buyer describes the work, sets a budget (ALGO) and deadline
3. **AI Negotiation** — Three AI seller agents (SpeedyDev, QualityCraft, BudgetPro) compete over 2 rounds, each with unique personality and pricing strategy. Messages generated by self-hosted LLM
4. **Winner Selected** — Optimizer scores all offers on cost, speed, and quality — picks the best deal with explanation
5. **Smart Contract Deployed** — A fresh ARC-4 escrow contract is deployed per deal with conditions (buyer, seller, amount, deadline)
6. **Escrow Funded** — Buyer signs a grouped transaction via Pera Wallet, locking ALGO into the smart contract
7. **AI Delivery Evaluation** — Seller AI generates a delivery update, then an Oracle AI evaluates it autonomously
8. **Funds Released or Refunded** — If delivery is accepted: ALGO released to seller. If rejected: ALGO refunded to buyer. All via smart contract inner transactions

## Running the App

### 1. Clone the repo

```bash
git clone <repo-url>
cd Accord
```

### 2. Set up the backend

```bash
cd backend
npm install
```

Create a `.env` file:

```bash
cp .env.example .env
```

Fill in the values:

```env
PORT=3001
ORACLE_MNEMONIC=<your 25-word algorand mnemonic>
ALGOD_SERVER=https://testnet-api.algonode.cloud
ALGOD_TOKEN=
ALGOD_PORT=443
NEGOTIATION_BACKEND_URL=http://165.245.134.230:9000/negotiate
```

Start the backend:

```bash
npm start
```

Backend runs on `http://localhost:3001`. Verify with `http://localhost:3001/health`.

### 3. Set up the frontend

Open a new terminal:

```bash
cd frontend
npm install
npm run dev
```

Frontend runs on `http://localhost:5173`.

### 4. Set up Pera Wallet

1. Open Pera Wallet app on your phone
2. Go to **Settings > Developer Settings > Node** and switch to **TestNet**
3. Create or import a TestNet account
4. Fund it using the [Algorand TestNet Faucet](https://bank.testnet.algorand.network/)

### 5. Use the app

1. Open `http://localhost:5173` in your browser
2. Click **Connect Wallet** — scan QR with Pera app
3. Fill in a task description, budget, and deadline
4. Watch the AI negotiation unfold
5. Review the winner and click **Fund Escrow**
6. Sign the transaction in Pera app
7. Watch the AI delivery oracle evaluate and auto-release/refund

## Smart Contract — DealEscrow

Written in Python using Puya, compiled to TEAL, deployed on Algorand TestNet.

**State machine:** `empty(0) → created(1) → funded(2) → verified(3) → released(4) / refunded(5)`

| Method | Description |
|--------|-------------|
| `createDeal(buyer, seller, amount, deadline)` | Oracle sets deal conditions |
| `fundDeal(payment)` | Buyer locks ALGO into escrow |
| `verifyComplete()` | Oracle marks delivery verified |
| `releaseFunds()` | Inner transaction pays seller |
| `refundBuyer()` | Inner transaction refunds buyer |

A fresh contract is deployed per deal — no state conflicts between deals.

## Key Design Decisions

- **No backend holds funds** — ALGO goes directly into the smart contract app account via Pera-signed transactions
- **AI-to-AI pipeline** — LLM negotiates, LLM evaluates delivery, smart contract executes the outcome
- **Per-deal contracts** — Each deal gets its own deployed contract to avoid state conflicts
- **Self-hosted LLM** — Custom middleware on AMD GPU droplet with vLLM for inference, falls back to Claude API
- **Visible intelligence** — Every AI message, score, and oracle reasoning is shown to the user
- **Visible trust** — Transaction hashes and Pera Explorer links at every step
