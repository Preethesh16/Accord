# Accord вЂ” Trustless Deal Engine with AI Negotiation

A trustless autonomous deal broker for freelance and service agreements. Buyers fear poor delivery, sellers fear non-payment. Accord replaces platform trust with AI negotiation powered by Claude API, Algorand smart contract escrow where no backend ever holds money, and automated verification and release.

The product feels like an AI agent that negotiates the best possible freelance deal, then executes the agreement trustlessly on-chain. Intelligence is visible. Blockchain trust flow is visible.

---

## How It Works

1. **Buyer connects** Pera Wallet on Algorand TestNet
2. **Buyer submits** a task description, budget (in ALGO), and deadline
3. **Three AI seller agents** enter a negotiation chat over 2вЂ“3 rounds вЂ” each message generated by Claude API in that seller's unique personality
4. **Optimizer scores** all final offers based on cost, speed, and quality вЂ” picks the winner and explains why
5. **Buyer signs** a grouped Algorand transaction via Pera that locks funds into smart contract escrow
6. **Backend oracle** calls `verifyComplete` on the contract (simulating delivery verification)
7. **Smart contract releases** ALGO to seller via inner transaction (or refunds buyer) вЂ” final tx hash shown in UI

## Build Plan

### Phase 1 вЂ” Smart Contract
Write and deploy the Puya ARC-4 escrow contract to Algorand TestNet. This produces the `APP_ID` that both backend and frontend depend on.

**Contract methods:**
- `createDeal` вЂ” initialize deal with buyer, seller, amount, deadline
- `fundDeal` вЂ” lock ALGO into the app account
- `verifyComplete` вЂ” oracle marks delivery as verified
- `releaseFunds` вЂ” inner transaction pays seller
- `refundBuyer` вЂ” inner transaction returns funds to buyer

### Phase 2 вЂ” Frontend UI (with mock data)
Build all visual components with hardcoded data so the look and feel is right before any backend logic exists. This includes wallet connection, buyer form, negotiation chat with typing animations, deal summary, escrow status, and the 7-step status timeline.

### Phase 3 вЂ” Backend (local first)
Build the Express server locally on `localhost:3001`. Wire up seller profiles, Claude API negotiation rounds, optimizer scoring, Algorand transaction building, oracle calls, and Redis state management. Test everything end-to-end locally before deploying.

### Phase 4 вЂ” Integration & Deploy
Replace mock data in the frontend with real API calls. Connect Pera Wallet for real transaction signing. Run real Claude API negotiations flowing into the chat UI. Execute real escrow transactions on TestNet. Once stable, deploy backend to Render and swap `VITE_BACKEND_URL` to the production URL.

## Key Design Decisions

- **No backend holds funds** вЂ” ALGO goes directly into the smart contract app account
- **Local-first development** вЂ” backend runs locally before any cloud deployment
- **AI intelligence is visible** вЂ” every negotiation message and optimizer reasoning is shown to the user
- **Blockchain trust is visible** вЂ” transaction hashes and AlgoExplorer links at every step
- **Scoring formula** вЂ” `0.4 * (1/price) + 0.3 * (1/days) + 0.3 * quality`
